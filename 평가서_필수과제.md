# 결과물에 대한 평가 — 필수과제 (basic)

## 1. 구현 목표

이번 과제에서는 단순히 요구사항을 화면에 띄우는 것에서 그치지 않고, **HTML로 구조를 잡고 CSS로 꾸미고 JavaScript로 동작을 붙이는 흐름**을 하나의 페이지 안에서 직접 연결해 보는 것을 목표로 했다. 특히 다음 세 가지를 스스로에게 기준으로 삼았다.

- 요구사항 체크리스트를 **하나도 빠짐없이** 충족할 것
- 동작만 되는 게 아니라 **왜 그렇게 짰는지 설명할 수 있는 코드**를 쓸 것
- 만든 뒤에는 **교안의 테스트 항목으로 직접 검증**하고, 오류가 나면 원인을 분석해 고칠 것

---

## 2. 결과물에 대한 평가

네개의 과제를 구현하면서 "동작만 되는 코드"가 아니라 표준 문법과 접근성을 지키는 코드를 쓰려고 했다. 과제별로 신경 쓴 핵심은 다음과 같다.

- **회원가입**: `required`·`pattern`·`minlength`만으로 자바스크립트 없이 브라우저가 검증·차단한다는 점을 이해하고, `<fieldset>`/`<legend>`로 선택 항목을 그룹화했다. 제출값은 `method=get` → 결과 페이지에서 `URLSearchParams.getAll()`로 복수 체크박스까지 표로 표시했다.
- **여행지**: `header/nav/main/section/article/aside/footer` 시맨틱 구조를 익히고, 고정 내비 + 앵커(`#id`) + `scroll-behavior:smooth`로 페이지 내 이동을 구현했다.
- **상품 갤러리**: `:root` 변수와 `var()`로 테마를 관리하고, Grid로 카드 배치·Media Query로 반응형(3→2→1열)·`@keyframes`로 등장 애니메이션·다크모드 토글을 구현했다.
- **할 일 앱**: 데이터를 `{ id, text, done }` 객체 배열로 관리하고, 이벤트 위임·모듈 분리·`localStorage` 영속화·비동기 `fetch`까지 적용했다.

목표로 삼았던 "요구사항 완전 충족 + 설명 가능한 코드"는 대체로 달성했다고 생각한다. 특히 CSS·JS를 각각이 아니라 **하나의 동작하는 페이지로 엮는 감각**을 얻은 것이 가장 큰 수확이다.

---

## 3. 구현하면서 어려웠던 점

- **모듈이 더블클릭으로는 안 열렸다.** 할 일 앱을 `type="module"` + `import/export`로 나눴더니, HTML 파일을 그냥 더블클릭하면 콘솔에 CORS 오류가 뜨며 아무것도 안 나왔다.
- **체크박스를 여러 개 골랐는데 결과에 하나만 나왔다.** 회원가입 관심분야를 복수 선택했는데 결과 페이지에는 첫 번째 값만 표시됐다.
- **고정 헤더가 본문 첫 줄을 가렸다.** 헤더를 `position:fixed`로 고정하니 스크롤 맨 위에서 본문 제목이 헤더 뒤로 숨었다.

---

## 4. 문제 해결 과정 (트러블슈팅)

### ① 모듈이 더블클릭으로 안 열리는 문제
- **문제**: 할 일 앱을 파일로 더블클릭하니 콘솔에 CORS 오류 → 화면이 비어 있음
- **원인 분석**: ES 모듈(`import`/`export`)은 `file://` 프로토콜에서 브라우저 보안 정책(CORS)으로 차단된다는 것을 알게 됨
- **해결**: 폴더에서 `python3 -m http.server 5500`으로 로컬 서버를 띄우고 `http://localhost:5500`으로 접속
- **배운 점**: 모듈·`fetch`를 쓰는 과제는 **실행 환경(http)** 자체가 요구사항의 일부다. README에도 "서버로 열 것"을 명시했다.

### ② 복수 체크박스가 하나만 넘어오는 문제
- **문제**: 관심분야를 2개 체크했는데 결과 표에 1개만 표시됨
- **원인 분석**: 값을 가져올 때 `querySelector`/`get()`을 쓰면 **첫 번째 요소만** 반환된다는 것을 확인
- **해결**: 같은 `name`의 값은 `URLSearchParams.getAll("interest")`로 배열째 받아서 표에 이어 붙이도록 수정
- **배운 점**: 여러 개를 처리해야 할 때는 "하나만 가져오는 API"와 "전부 가져오는 API"를 구분해서 써야 한다.

### ③ 고정 헤더가 본문을 가리는 문제
- **문제**: `position:fixed` 헤더가 스크롤 최상단에서 본문 제목을 덮음
- **원인 분석**: `fixed`는 요소를 문서 흐름에서 빼내 위에 겹치므로, 그 높이만큼 본문이 밀려 올라옴
- **해결**: 본문(또는 body)에 헤더 높이만큼 `padding-top`을 주어 겹침을 방지
- **배운 점**: `fixed`는 "떠 있는" 요소라 항상 **자리 보정**을 함께 생각해야 한다.

## 5. 코드에서 가장 신경 쓴 부분

- **의미 있는 변수·함수명**: `fetchQuote()`, `createTodo()`, `saveTodos()`처럼 이름만 봐도 역할이 보이게 지음.
- **관심사 분리**: 할 일 앱을 데이터 계층(`storage.js`)과 화면 계층(`app.js`)으로 나눠, 데이터 로직을 바꿔도 화면 코드는 건드리지 않게 함.
- **이벤트 위임**: 할 일 항목마다 리스너를 다는 대신 부모 `ul`에 리스너 1개를 걸고 `closest()`로 대상을 찾아, 동적으로 추가된 항목도 자동 처리되게 함.
- **CSS 변수(:root)**: 색·간격을 한 곳에서 정의하고 `var()`로 소비 → 다크모드를 값 교체만으로 전환.
- **예외 처리**: 비동기 `fetch`를 `try/catch`로 감싸고, 실패 시 기본 문구로 폴백해 화면이 깨지지 않게 함.
- **접근성**: 모든 `img`에 의미 있는 `alt`, `label`–`input` 연결, `prefers-reduced-motion`으로 모션 최소화.

## 6. 테스트 및 검증 결과

교안의 "테스트 방법" 항목을 그대로 기준으로 삼아 직접 검증했다.

### 회원가입 & 결과 페이지
| 테스트 | 기대 결과 | 실제 결과 |
|---|---|---|
| 아무것도 입력 없이 제출 | 브라우저가 제출 차단 | ✅ 차단됨 |
| 아이디 2자 입력 | 패턴(4~12자) 위반 경고 | ✅ 경고 |
| 비밀번호 3자 입력 | 8자 이상 안내 | ✅ 안내 |
| 관심분야 2개 체크 → 제출 | 결과에 두 값 모두 표시 | ✅ 함께 표시 |
| 결과 페이지 스크롤 | 상단 헤더 고정 유지 | ✅ 고정 |

### 여행지 소개 페이지
| 테스트 | 기대 결과 | 실제 결과 |
|---|---|---|
| 내비 '먹거리' 클릭 | 해당 섹션으로 부드럽게 스크롤 | ✅ smooth 이동 |
| 아래로 스크롤 | 상단 내비 고정 | ✅ 고정 |
| 명소 카드 hover | 이미지 살짝 확대(zoom) | ✅ scale |
| 푸터 '↑ 맨 위로' | 최상단 이동 | ✅ 이동 |
| 영상 재생 | controls로 재생/정지 | ✅ 동작 |

### 상품 카드 갤러리
| 테스트 | 기대 결과 | 실제 결과 |
|---|---|---|
| 폭 1200→800→500px | 3→2→1열, 500px에서 메뉴 숨김 | ✅ 반응형 |
| 다크 버튼 | 배경·카드·글자색 한 번에 전환 | ✅ 전환 |
| 카드 hover | 위로 떠오르고 그림자 강화 | ✅ translateY |
| 새로고침 | 히어로 문구 순차 등장 | ✅ @keyframes |
| OS 동작 줄이기 ON | 애니메이션 비활성화 | ✅ 접근성 대응 |

### 할 일 관리 앱
| 테스트 | 기대 결과 | 실제 결과 |
|---|---|---|
| 로컬 서버로 열기 | 모듈 정상 로드 | ✅ (더블클릭은 오류) |
| 입력 후 Enter | 목록 추가 + 입력창 비워짐 | ✅ |
| 체크박스 클릭 | 취소선 + 완료 개수 +1 | ✅ |
| '완료' 필터 | 완료 항목만 표시 | ✅ |
| 항목 추가 후 새로고침 | localStorage로 유지 | ✅ 영속 |
| 오늘의 한마디 | fetch 로드(차단 시 기본 문구) | ✅ 폴백 동작 |

## 7. 개선 및 추가 구현 계획

- **회원가입**: 이메일/아이디 중복 확인, 비밀번호 강도 표시
- **여행지**: 실제 날씨/지도 API 연동, 이미지 확대 보기
- **갤러리**: 검색·정렬 기능, 장바구니
- **할 일**: 마감일·우선순위·카테고리, 드래그 정렬

> 위 아이디어 중 일부는 실제로 추가과제(`premium/`)에서 구현했으며, 해당 내용은 **추가과제 평가서**에 별도로 정리했다.

### 회고
단순히 요구사항을 구현하는 데서 끝내지 않고, 실제 사용자가 이용하는 상황을 고려해 입력 검증·예외 처리·가독성·유지보수성을 함께 고려하려 했다. 구현 후에는 교안의 테스트 항목을 기준으로 모든 기능을 직접 검증했고, 오류가 난 부분은 원인을 분석해 수정한 뒤 다시 테스트했다. 구현 → 검증 → 수정을 반복하며 "동작하는 코드"에서 "이해하고 설명할 수 있는 코드"로 나아간 과정이 이번 실습의 가장 큰 배움이었다.
